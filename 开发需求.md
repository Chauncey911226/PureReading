# Pure Reading 初赛 MVP 开发需求文档

## 1. 项目概述与架构设计

**Pure Reading** 是一款专为降低长辈阅读门槛设计的“适老化纯净阅读引擎”。
本阶段（MVP）追求绝对的极简，**系统内不包含任何用户鉴权、登录体系，也不使用任何数据库进行持久化存储**。前后端的数据流转完全依赖 URL 参数（如编码后的字符串）进行隐式、无状态的传递。

**核心架构：唯一端沉浸式 SPA（单页应用）**
全站只有一个核心页面入口。系统需通过侦听 URL 中是否携带目标链接参数，自动在以下两种形态间平滑切换：
* **输入态（初始无参数）**：大面积留白，屏幕中央仅展示极简的网址输入表单与解析按钮。
* **阅读态（带有隐式目标参数）**：输入表单自动退场，全屏沉浸式渲染提纯后的大字号正文，同时从屏幕右侧弹出AI 摘要弹窗。

## 2. 核心交互工作流与体验边界

**【步骤一：解析请求与状态反馈】**
* 用户粘贴目标网页链接并提交后，前端需立即进入防连点的加载状态。
* **视觉反馈**：必须提供明确的适老化加载提示（如骨架屏或清晰的文本提示，告知正在剥离广告）。
* **状态流转**：前端需将原始长链接安全编码后，静默更新至当前浏览器的地址栏，确保持当前页面的 URL 可以被直接复制和分享，且任何人打开该 URL 都能直接进入对应的阅读态。

**【步骤二：全端自适应阅读态原地渲染 (Reader Mode)】**
当后端处理完毕返回数据后，页面进入核心的适老阅读态，必须严格遵守以下跨端体验边界：

1. **纯净正文区（响应式适老排版）**：
    * **色彩方案**：必须采用高对比度的护眼配色（例如暖色背景配合极深的文字色），满足无障碍阅读标准。
    * **大屏防拉伸**：在电脑或平板端，必须为正文内容区设定合理的最大宽度并居中，坚决避免文本随屏幕无限横向拉伸导致长辈视线追踪困难。
    * **移动端流式自适应**：在手机端，必须确保正文拥有极其宽裕的安全内边距（防误触）。字号必须采用流式缩放策略，保证在任何尺寸的手机上，基础字号都足够大（符合老年人视力需求）且能优雅地自动换行。严禁出现需要长辈左右拖拽或双指放大的情况。
    * **绝对防误触红线**：必须在前端逻辑或样式层面，彻底抹除正文内容中所有原有超链接的视觉伪装（如下划线、主题色）以及点击跳转行为。把该页面变成一个无论怎么点都不会跳出的“安全屋”。

2. **右侧 AI 摘要弹窗 (Bottom Sheet)**：
    * 渲染为屏幕右侧的悬浮卡片层，内部醒目排布 AI 提炼的白话要点。
    * **交互规约**：具备可视化的拖拽把手。当首次进入阅读态时，该弹窗必须默认自动展开至合适尺寸，确保长辈第一眼就能看到大白话总结；随后允许用户向右滑动收起以阅读正文。

3. **分享闭环胶囊 (Action Bar)**：
    * 在界面的醒目位置（如右下角悬浮或顶部固定）提供“一键复制专属链接”的分享入口。
    * 点击后需自动将当前浏览器地址栏的完整状态（包含解析参数）写入系统剪贴板，并弹出语气温暖、明确的成功提示（Toast），引导子女去微信粘贴转发。

## 3. 引擎大脑：API 编排与大模型指令

前端不可直接暴露核心 API 密钥，需通过轻量级后端服务（如 Serverless 云函数）串行执行以下动作，并整合返回给前端：

**【动作一：净网文本抽取】**
* 调用可靠的网页纯文本提取服务（如 Jina Reader 等同类工具），从目标 URL 中剥离所有 HTML 结构、侧边栏和广告干扰，仅提取干净的核心内容文本。

**【动作二：大模型降维翻译】**
* 将提取到的纯净文本发送给大模型处理。
* **人设与业务提示词边界**：
    1. 角色：极具耐心的社区助老志愿者。
    2. 任务：将硬核长文提炼为 3 到 5 句长辈能轻松听懂的“大白话”要点。
    3. 风格约束：语气必须有温度且带有安抚性。若原文包含行动指南（如报销流程、防骗预警），摘要中必须明确给出“接下来该怎么做”的行动建议。
    4. 禁忌词汇：绝对禁用任何书面化、公文式的词汇（如“该文章指出”、“据统计”），必须转换为生活化口语。
* **数据契约交付**：大模型及后端服务必须以**结构化的格式**（如 JSON）返回最终结果，明确区分出“白话摘要列表”与“排版完好的纯净正文”两部分独立数据，供前端按需渲染。
